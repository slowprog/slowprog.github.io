</a>Большая часть начинающих блогеров начинают с WordPress, и конечно же всем хочется расширить стандартный функционал. Первое, что приходит в голову начинающему блогеру - "Плагины!". Но самое легкое и очевидное решение не всегда является лучшим. В этом случае это именно так. Много плагинов - это много проблем. Каждый плагин создает небольшую нагрузку на сервер, посылает лишние запросы в базу данных и занимает места больше, чем нужно. И если плагинов много, то эти показатели вырастают в разы, что грузит сервак, какое хорошее бы у вас кэширование не стояло.

И лично моя позиция в том, чтобы избавляться от ненужных и лишних плагинов. Редкий плагин заслуживает безоговорочного присутствия на блоге. В этом посте я постараюсь рассказать, какие плагины нужно точно удалить, а какие можно и оставить. Приступим:

  • Scroll to Top Button (и все подобные) — плагин для создания на блоге кнопки Наверх</em> для прокрутки контента обратно. Бесполезный, так как я могу предложить свой настраиваемый код без плагина Scroll to top без плагина для WordPress</a>.</li>
  • HTML Page Sitemap (и все подобные) — плагин для создания страницы с картой сайта в HTML формате. Бесполезный, так как я могу предложить свой настраиваемый код без плагина HTML карты сайта для WordPress</a>.</li>
  • Google XML Sitemaps - плагин для создания XML карты блога. Но плагин, довольно сильно нагружает блог, потому я написал пост по настройке Google XML Sitemaps</a>, чтобы снизить нагрузку. А в том случае, если вам этот богатый и тяжёлый функционал вообще не нужен, то прошу используйте мой генератор XML карты сайта для WordPress</a> без плагина.</li>
  • Top Contributors (и все подобные) — плагин для создания ТОПа комментаторов. Бесполезный, так как я могу предложить свой настраиваемый код без плагина Топ комментаторов без плагина в WordPress</a>.</li>
  • Akismet - плагин для отсеивания спам-комментариев. Плох тем, что для работы своей использует сторонний ресурс. Я уже писал про защиту от спама без плагина</a>.</li>
  • Различные плагины для создания форм обратной связи. Обычно генерация форм занимает лишние ресурсы. Я рекомендую форму обратной связи без плагина</a>.</li>
  • PHP Code Widget - плагин для того, чтобы в виджетах исполнялся PHP-код. Тут плагин совсем ни к чему, т.к. функция исполнения PHP-кода в виджете очень простая. Подробней читайте в статье PHP в виджете WordPress</a>.</li>
  • WordPress Related Posts (и все подобные) — плагин для вывода похожих записей на блоге. Куча написано по этому поводу, и можно заменить плагин совсем небольшим кодом.</li>
  • WP Page Numbers (и все подобные) — плагин для вывода постраничной навигации в блоге. Фактически бесполезен, т.к. этот функционал есть в самом WordPress.</li>
  • Disable WordPress Updates (и все подобные) — плагин для отключения автоматических обновлений WordPress'а и плагинов. Всего несколько строк кода отключают эту возможность.</li>
  • Breadcrumbs Everywhere (и все подобные) — плагин для создания хлебных крошек на блоге. Все просто решается без плагина.</li>
    </ul>
    Это список плагинов, которые точно не нужны и можно смело удалять, кстати, не забудьте удалить таблицы, которые останутся после удаления плагинов. И вообще, удаляйте все плагины, которые могут быть заменены обычным кодом или просто бесполезны (или польза минимальна и не стоит нагрузки на сервер). И не забудьте мне сообщить в комментариях, какой вы решили удалить плагина и почему, чтобы я пополнил список на благо будущих поколений.

    А какие же плагины могут понадобится? Я посоветую всего некоторые из них:

    • Redirection</a></strong> - плагин для организации перенаправления по редиректу. Может пригодится при удалении части контента или смене алгоритма организации URL.</li>
    • Rus-To-Lat</a></strong> - плагин для автоматической генерации ЧПУ на латинице, а то кириллица в некоторых браузерах выглядит как абракадабра в urlencode.</li>
    • Subscribe To Comments</a></strong> - плагин для организации возможности подписываться на комментарии к конкретному посту.</li>
    • WP-Optimize</a></strong> - плагин для оптимизации базы данных. Можно иногда активировать, запускать и после снова деактивировать.</li>
    • Platinem SEO Pack</a></strong> - плагины для помощи в SEO оптимизации каждого поста.</li>
    • Pretty Link</a></strong> - плагин для маскировки ссылок и переделки внешних ссылок во внутренние.</li>
    • W3 Total Cache</a></strong> - плагин для организации тотального кеширования каждого аспекта WordPress. Достаточно сложен и потому я написал пост W3 Total Cache настройка</a> для того чтобы вам облегчить понимание его работы.</li>
      </ul>
      Вот примерно такие плагины вам точно понадобятся, для того чтобы сделать ваше администрирование блогом легче и удобнее.

      На этом у меня все. Если есть какие-то мысли по поводу добавления плагинов в один из списков, то я вас внимательно слушаю в комментариях. Какие плагины вы на своем блоге считаете незаменимыми? Всем удачи!
