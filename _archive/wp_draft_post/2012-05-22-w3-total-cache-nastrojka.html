    </a>

    Вкладка Minify</h2>
    Идем дальше по вкладкам. Следующая после Page Cache</em> идет Minify</em>. На этой вкладке можно настроить более детально параметры уменьшения HTML, JS и CSS. Тут так же все делиться на отдельные блоки и каждый мы разберем.

    General</strong> — блок общих настроек для минимизации файлов. Тут отметьте все галочками, чтобы перезаписывать структуру URL и отключить минимизацию для зарегистрированных пользователей. Кроме того, в выпадающем списке вы можете установить уведомления об ошибках при минимизации файлов, а можете не устанавливать.
    </a>
    HTML & XML</strong> — блок настроек для минимизации файлов HTML и XML. Я рекомендую отметить все галочки кроме Don't minify feeds</em>. Тогда у вас будут сжаты сами файлы HTML и XML, все стили и JS написанные в коде страницы, а так же содержимое RSS канала. Окошко внизу предназначено для возможности оставить комментарии на странице, т.к. минимизация убирает все комментарии, кроме тех, в тексте которых содержаться слова указанные в этом окошке.
    </a>
    JS</strong> — блок настроек для минимизации файлов JavaScript. Как я уже говорил выше, минимизация файлов JavaScript может привести к ошибкам со слиянием строк, так что используйте это с осторожностью. У меня были небольшие проблемы, но я быстро их устранил. Для минимизации отметьте галочки как на картинке, если только вы не хотите объединить весь JS из разных концов страницы (например, слить файлы из хедера и футера). Так же вы можете отдельно добавлять файлы JS с помощью кнопки Add a script</em>. Но перед этим в выпадающем списке выше выберете вашу тему.
    </a>
    CSS</strong> — блок настроек минимизации файлов стилей. В принципе настройки такие же, как и в блоке выше, можете настроить как на картинке и нормально. Поэтому по второму разу рассказывать не буду, кроме свойства @import handling</em>. С помощь него можно применить правило @import</em> в файлах CSS, оно позволяет импортировать содержимое одного CSS-файла в другой. В выпадающем списке выберете Process.
    </a>
    Advanced</strong> — блок дополнительных настроек минимизации файлов. Тут можно ничего не трогать и оставить все как есть. А вообще, тут можно изменить время обновления кэша внешних файлов, время сбора мусора, указать юзер агенты, которым не отдавать кэш и страницы, которые не должны быть сжаты.
    </a>

    Вкладка Database Cache</h2>
    Идем дальше по вкладкам. Следующая после Minify</em> идет Database Cache</em>. На этой вкладке можно настроить чуть более детально кэширование базы данных. Фактически здесь и настраивать нечего.

    General</strong> — блок общих настроек кеширования базы данных. Тут только проверьте, чтобы единственная голочк была поставлена. Она отвечает за то, чтобы не кэшировать запросы для зарегистрированных пользователей. С ней вы будете видеть актуальные страницы вашего сайта, когда будете сидеть под админом.
    </a>
    Advanced</strong> — блок дополнительных настроек кеширования базы данных. Опять же, тут возится не с чем. Как и в других дополнительных настройках, вы можете установить время сбора мусора, время жизни кэша, страницы, которые надо выкинуть из кэша и написать словоформы для игнорирования запросов (запросы, которые будут содержать эти слова не будут кэшироваться).
    </a>

    Вкладка Object Cache</h2>
    Идем дальше по вкладкам. Следующая после Database Cache</em> идет Object Cache</em>. На этой вкладке, собственно, настраивать нечего, потому что, как я ранее говорил, этот вид кэширования может быть совершенно бесполезен. Но если вы все таки его установили, то все настройки можно оставить как есть и ничего не трогать.

    Вкладка Browser Cache</h2>
    Идем дальше по вкладкам. Следующая после Object Cache</em> идет Browser Cache</em>. На этой вкладке можно довольно подробно настроить кэширование страницы браузером, фактически мы можем указать браузеру что и на сколько кэшировать.

    General</strong> — блок общих настроек для кэша браузера. По сути, тут все просто и вам надо сделать так же как на рисунке: все выделить кроме двух последних. Т.к. последние два не нужны и будут нагружать сервер лишними действиями.
    </a>
    CSS & JS</strong>, HTML & XML</strong>, Media & Other Files</strong> — блоки настройки кэширования браузером файлов JS, CSS, HTML, XML и других. Тут все так же просто и можно оставить, как есть. Единственное, что вы может захотите изменить это время истечения срога существования кэша в каждом из блоков. У себя я установил следующие цифры: для CSS и JS 2628000 секунд (1 месяц), для HTML и XML 3600 секунд (1 час) и для остальных файлов 31536000 секунд (1 год). У себя вы можете выставить и другие цифры, тут надо исходить из того, как часто вы собираетесь менять эти файлы на своем сайте. Если это происходит редко, то большие цифры можно поставить везде.
    </a>

    Заключение</h2>
    После настройки кэширования необходимо обновить сам кэш. Вверху появится кнопка Empty page cache</em>, вот ее то и нужно нажать для очистки кэша от устаревших файлов. После этого вы увидите вверху зеленое слово Enable</em>, означающее, что плагин успешно функционирует.

    Настройка довольно долгая и большая, возможно все не выйдет с первого раза и в процессе будут какие-то проблемы, но оно того стоит. Проверил на своей шкуре.
