Довольно многие нынче его используют для разработок и обучения, потому, я думаю, это целесообразно. В прошлый раз я писал о том как установить Symfony 2</a> в общих чертах. Если кому-то этого не хватило для того, чтобы поставить Symfony на Denwer самостоятельно, то читайте дальше - тут все написано и описано. Постараюсь все изложить четко и ясно, предупредив все ваши возможные ошибки.

Для начала было бы неплохо определиться, что нам понадобится скачать с Инета для нормальной установки фреймворка:

  • Скачать стандартный пакет Denwer</a> (на момент написания статьи самая свежая сборка 2010-11-07).</li>
  • Скачать дополнительные модули для Denwer</a>, чтобы были недостающие расширения для PHP . Эти расширения необходимо для корректной работы Symfony.</li>
  • Скачать фреймворк Symfony 2</a> (на момент написания статьи версия 2.0.4). Я скачал сборку без вендоров (without vendors).</li>
  • Скачать распределенную систему управления версиями Git</a>. Там справа окошко, где нужно ткнуть в необходимую вам операционку и на следующей странице будет список с релизами, там то и скачайте нужный. Git нужен как для установки, так пригодится и для дальнейшей разработки.</li>
    </ul>
    Это все, что нам понадобится для установки Symfony2 на Denwer. Теперь можно приступать непосредственно ближе к делу:

    • Устанавливаем Denwer куда вам удобнее. Для примера предположим, что я его поставил по адресу c:/localsites/denwer</em>.</li>
    • Устанавливаем скачанное расширение для Denwer. Вместе с ним в PHP устанавливаются нужные нам дополнительные модули. Я конечно точно не знаю нужны ли все эти расширения для успешной работы Symfony2 (не проверял), но у меня они были установлены и все работало прекрасно. Их необходимо включить в конфигурационном файле PHP по адресу c:/localsites/denwer/usr/local/php5/php.ini</em>, для этого найдите и расcкомментируйте, если надо, следующие строки:
      [cce lang="text"]extension=php_gd2.dll
      extension=php_interbase.dll
      extension=php_mbstring.dll
      extension=php_mysql.dll
      extension=php_mysqli.dll
      extension=php_pdo_mysql.dll
      extension=php_pdo_pgsql.dll
      extension=php_pdo_sqlite.dll
      extension=php_pgsql.dll
      extension=php_sqlite.dll
      extension=php_xsl.dll[/cce]
      Обязательно проверьте на месте ли все расширения, зайдите в c:/localsites/denwer/usr/local/php5/ext</em> и проконтролируйте, чтобы все что раскомментировали было в этой папке. В противном случаи, попробуйте переустановить расширение для Denwer.</li>

    • Не забываем внести путь к бинарнику php</strong> в переменные окружения. Для этого в свойствах системы находим вкладку Дополнительно</em> и там внизу жмем на кнопку Переменные среды</em>. В переменную Path</em> необходимо дописать, через точку с запятой (если там уже есть что-нибудь), путь к файлу php.exe. В моем случаи путь до PHP выглядит следующим образом: c:/localsites/denwer/usr/local/php5/</em>.</li>
    • Ставим Git. Надеюсь там все справятся, в принципе, там только Далее</em> нажимать и придерживаться подсказок инсталлера. Только когда вас спросят о том как вы хотите использовать Git и будите ли модифицировать глобальную переменную PATH (Adjusting your PATH environment</strong>), то отметьте второй вариант Run Git from the Windows Command Promt</strong> и нажмите Далее</em></li>
    • Настало время установить сам Symfony 2. Сделаем это в стандартной директории для Denwer. Проходим в c:\localsites\denwer\home</em>, создаем там папку mysite.ru</em>, а в ней еще папку www</em>. Первая папка ограничивает корневую директорию для конкретного сайта, а из второй папки Денвер запускает сам сайт на исполнение, то бишь туда размещают файлы самого сайта. После всех этих действий получается следующий путь к нашему сайту c:\localsites\denwer\home\mysite.ru\www\</em>. Распаковываем туда наш скачанный архив с Symfony2 и получаем следующую иерархию:
      [cce lang="text"]mysite.ru/ <- корневая директория сайта
      www/ <- папка с исполняемыми файлами
      Symfony/ <- распакованный архив
      app/ <- директория с конфигурацией проекта
      ...
      src/ <- директория со всем PHP-кодом проекта
      ...
      vendor/ <- директория с подключаемыми библиотеками
      ...
      web/ <- видимая часть сайта, содержит javascript, стили и т.д.
      ...[/cce]
      </li>

    • Запускаем командную строку и переходим в папку проекта с помощью команды cd</em>. В моем случаи команда выглядит так:
      [cce lang="text"]cd /d c:/localsites/denwer/home/mysite.ru/www/Symfony[/cce]
      </li>

    • Поскольку дистрибутив Symfony скачивали без вендеров настала пора их установить. Именно тут нами понадобится Git, но мы его не будем запускать сами, его запустит команда, которую мы введем ниже. Так же для этого потребуется подключение к интернету, если его нет, то надо было качать с вендорами и можно пропустить этот шаг. Для того, чтобы установить вендоры необходимо запустить следующу команду на выполнение в консоле:
      [cce lang="text"]php bin/vendors install[/cce]
      Эта команда загружает все сторонние библиотеки (включая сам Symfony) в папку vendor/</em>. Очень удобно, не правда ли? Вы можете переносить проект в любое место и прямо оттуда обновлять Symfony.</li>
      </ul>
      Всё, теперь сервер готов и Symfony2 установлен. Можно приступать к настройкам самого Symfony с помощью визуального тестера конфигурации сервера. Он призван помочь убедиться в том, что ваш веб-сервер и PHP корректно настроены на использование Symfony. Для того, чтобы запустить проверку наберите в браузере следующий адрес:
      [cce lang="text"]http://mysite.ru/Symfony/web/config.php[/cce]
      На этой странице вы увидите результаты проведенного анализа. Там будут рекомендации по исправлению недочетов и неисправностей, если такие будут присутствовать. Какие-то из них обязательны (MAJOR PROBLEMS) для исполнения, а другие только рекомендованы (RECOMMENDATIONS) для улучшения работы Symfony. Под каждым пунктом будет сказано, что вам необходимо установить, чтобы Symfony начал функционировать. В моем случаи почти никаких проблем не возникло и предварительных настроек сервера хватило, чтобы Symfony нормально функционировал. Высветилось только 3 рекомендации, но я поленюсь и исправлю только одну связанную с короткими тегами для кода PHP (Set short_open_tag to off in php.ini), потому что предпочитаю с длинными тегами, так меньше бажности. Если же у вас возникли проблемы, то попытайтесь их исправить и повторить процедуру проверки. В противном случаи пишете в комментарии - попытаюсь помочь.

      Надеюсь проблем не возникло. Тогда можно переходить по ссылке Bypass configuration and go to the Welcome page</em>. Теперь вы можете начать разработку своего проекта.
